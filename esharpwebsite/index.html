Readable code with systems-grade control

Compare equivalent data pipeline implementations in E# and C#. E# keeps the expressive surface area while offering deterministic memory and inline assembly where needed.

E# version deterministic + asm 
namespace Analytics.Stream;
class Program {
    void Main() {
        span<int32> buffer = stackalloc int32[1024];
        auto readings = Sensor.ReadLatest(buffer);
        var total = readings
            .filter(v => v > 0)
            .map(v => v * 2)
            .sum();

        // Pin registers for hot path emission
        asm {
            mov rax, total
            mov rdi, readings.length
            call [EmitMetric]
        }

        Console.WriteLine($"avg(ms) = {readings.avg()}" );
    }
}
Memory is stack-bound, bounds-checked, and still grants explicit register directives.
C# version GC managed 
using System;
using System.Linq;

class Program {
    static void Main() {
        int[] buffer = Sensor.ReadLatest(1024);
        var filtered = buffer
            .Where(v => v > 0)
            .Select(v => v * 2);

        var total = filtered.Sum();
        Metrics.Emit(total, filtered.Count());

        Console.WriteLine($"avg(ms) = {filtered.Average()}" );
    }
}
Familiar syntax, but allocations and GC pauses are unavoidable for hot loops.
